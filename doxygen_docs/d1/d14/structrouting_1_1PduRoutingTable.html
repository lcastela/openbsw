Detailed Description

The input of the routing is the output of the adapters, i.e. Internal PDU IDs.

The routing table consists of two parts:

1) Routing to destinations. The internal PDU ID is used as index into destinationOffsets. The range destinations[destinationOffsets[pduId], destinationOffsets[pduId + 1][ are the destination channel indices to which the PDU is routed.

2) Translation from internal PDU ID to destination channel specific addressing. The index of a routing in the destinations table is also used in the outputMessageIds table. This contains the message ID to output on the channel for this message.

The sizes of the spans are expressed according to:

  • N: the number of known PDUs
  • M: the number of channels
